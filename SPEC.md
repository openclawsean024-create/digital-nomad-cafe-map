## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 數位牧民（遠距工作者） | ~5,000 | 在異地城市找不到適合工作的咖啡廳，常踩雷（吵/沒插座/限時） |
| 自由工作者 / 接案者 | ~30 萬 | 需要工作咖啡廳但不知哪裡 WiFi 穩、座位舒服 |
| 數位遊牧新手 | ~10 萬 | 規劃移動時想知道下一個城市的工作咖啡廳 |
| 咖啡廳業者 | ~5,000 | 想曝光給數位牧民社群 |

### 1.2 為什麼不做替代方案

- **Google Maps 搜尋「咖啡廳」**：資訊雜亂、沒有工作友善標籤
- **Workfrom / WiFi Tribe**：歐美為主、台灣資料少
- **Facebook 社團「咖啡廳」**：資訊分散、無結構化資料
- **我們的解圖**：互動地圖 + 工作友善評分（WiFi/座位/插座/吵雜/限時）+ 使用者評論，純前端

---

## 2. 解決方案

### 2.1 核心價值主張

> 「下一個城市哪裡可以工作 — 一張地圖就找到。」

### 2.2 使用者流程

1. 進入地圖（Leaflet + OpenStreetMap）
2. 搜尋城市或瀏覽地圖上的咖啡廳標記
3. 點選咖啡廳看詳細評分（WiFi/座位/插座/吵雜/限時 5 維度）
4. 看其他牧民評論
5. 提交新咖啡廳（含評分 + 評論）

---

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] Leaflet 互動地圖（OpenStreetMap）
- [ ] 咖啡廳搜尋（依城市）
- [ ] 5 維度評分（WiFi/座位/插座/吵雜/限時）
- [ ] 使用者評論 + 照片
- [ ] 提交新咖啡廳（含自動 geocoding）
- [ ] 篩選（WiFi 強 / 插座多 / 不限時）
- [ ] 我的最愛清單

### 3.2 v2（加值）

- [ ] Freemium 商業模式（咖啡廳業者付費曝光）
- [ ] 多語系（中英日）
- [ ] 路線規劃（多個咖啡廳）
- [ ] 社群挑戰（每月拜訪 N 間咖啡廳）

### 3.3 明確不做

- 咖啡廳訂位/預約（不涉入店家營運）
- 餐飲外送整合
- 評論審核過濾（純使用者自律 + 檢舉）
- 與 Google Maps 評分同步（避免依賴）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | Next.js 14 + TypeScript | SSR 友善 |
| 地圖 | Leaflet + OpenStreetMap | 免費、開源 |
| Geocoding | Nominatim (OSM 官方) | 免費 |
| 資料庫 | Prisma + PostgreSQL | 評論/評分多筆 |
| 圖片 | Cloudinary / ImgBB | 評論照片託管 |
| 部署 | Vercel + Railway |
